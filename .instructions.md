# ImpactPilot Project Instructions

## UI Configuration

### Step 1: Idea Entry Form

**Idea Type Dropdown Options** (impactpilot/ui.py):
- Software/Digital Tool
- Assay/Analytical Method
- Therapeutic/Drug
- Diagnostic
- Medical Device
- Platform Technology
- Process Optimization
- Other

**R&D Stage Dropdown Options** (impactpilot/ui.py):
- Discovery/Research
- Proof of Concept
- Preclinical
- Phase 1 Clinical
- Phase 2 Clinical
- Phase 3 Clinical
- Regulatory/Commercialization

These options are tailored for biotech/pharma R&D workflows and must be preserved across code changes.

### Step 2: Evidence Search

**Auto-generate PubMed Query**: When navigating to Step 2, the app automatically generates a PubMed search query from the idea title and description (if the query field is empty). This is a deterministic keyword extraction (not LLM-based) that removes stopwords and keeps the 10 most relevant terms.

## Core Requirements

### No-Fabrication Rule
The app must NEVER fabricate numeric assumptions. It may only populate numeric fields if numbers are explicitly extracted from selected PubMed abstracts. All missing assumptions must show "[Not found in selected evidence]".

### Disclaimer Text
The exact disclaimer text must be preserved:
```
Disclaimer: This tool generates directional estimates. You must validate all inferred assumptions and evidence before using TCO or productivity numbers in decisions, submissions, or commitments.
```

### Report Structure
Impact Briefs must contain exactly these sections in order:
1. Title + Disclaimer (quoted)
2. ## Summary
3. ## Evidence used
4. ## Evidence-derived assumptions
5. ## Computed TCO
6. ## Computed R&D pipeline productivity
7. ## Evidence mapping
8. ## Citations
9. ## Open questions

### Testing Requirements
All modules must be unit-tested with pytest. Any new calculation or report generation logic requires corresponding test coverage.
